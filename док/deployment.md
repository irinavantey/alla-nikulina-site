# Публикация сайта

## GitHub Pages

Репозиторий:

https://github.com/irinavantey/alla-nikulina-site

Настройки публикации:

- Settings -> Pages
- Source: `Deploy from a branch`
- Branch: `main`
- Folder: `/docs`
- Save

После публикации сайт должен открыться по адресу:

https://irinavantey.github.io/alla-nikulina-site/

## Почему используется папка `docs`

GitHub Pages для этого репозитория предлагает публикацию из:

- `/root`
- `/docs`

Рабочая версия сайта находится в `site/`, поэтому была создана публикационная копия `docs/`.

## Отличие `site/` и `docs/`

`site/`:

- основная версия для разработки;
- содержит ссылки на локальные видео;
- подходит для просмотра на компьютере и загрузки на обычный хостинг, если видео тоже загружать.

`docs/`:

- версия для GitHub Pages;
- не содержит тяжелых `.mp4`;
- видео-теги оставлены без `src`, чтобы не было битых локальных видеофайлов в Git.

## Что нужно сделать для полноценного видео на публичном сайте

Вариант 1: загрузить видео на внешний видеохостинг и заменить блоки `<video>` на embed.

Вариант 2: сжать видео и добавить легкие web-версии в репозиторий или на сервер.

Вариант 3: при загрузке на обычный хостинг использовать папку `site/` вместе с `site/assets/video/`.
